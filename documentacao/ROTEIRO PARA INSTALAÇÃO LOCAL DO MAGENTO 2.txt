INSTALAÇÃO E CONFIGURAÇÃO DO AMBIENTE
=====================================

INSTALAR O XAMPP
----------------


1) BAIXAR O XAMPP ATRAVÉS DO SITE: https://www.apachefriends.org/pt_br/download.html (7.2.30 / PHP 7.2.30)
   
2) INSTALAR O PHP, APACHE E MARIADB A PARTIR DO ARQUIVO BAIXADO DO XAMPP


INSTALAR O COMPOSER
-------------------

3) BAIXAR O COMPOSER ATRAVÉS DO SITE: https://getcomposer.org/download/

4) INSTALAR O ARQUIVO Composer-Setup.exe 

5) ENTRAR ATRAVÉS DO SHELL DO PAINEL XAMPP E ENTRAR NA PASTA htdocs

6) INSTALAR O COMPOSER ATRAVÉS DO SHELL NA PASTA htdocs COM O COMANDO: php composer-setup.php

7) EXECUTAR O Control Painel do XAMMP (Procure no menu de programas do windows e DEIXE ELE ABERTO)
   ATENÇÃO: EXECUTAR O XAMPP COMO ADMINISTRADOR

8) INSTALAR O COMPOSER PELO LOCAWEB:
   curl -sS https://getcomposer.org/installer | php7.3 -c /home/tedolls1/.user.ini
   php7.3 -c /home/tedolls1/.user.ini composer.phar



ALTERAR OS ARQUIVOS DE CONFIGURAÇÃO DO XAMPP
--------------------------------------------

1) ALTERAR O ARQUIVO php.ini NO BOTÃO Config DO SERVIDOR Apache no XAMPP.
   Liberar as seguintes linhas do arquivo:
   ;extension=soap          	--> extension=soap
   ;extension=xsl      		--> extension=xsl
   ;extension=intl     		--> extension=intl
   ;disable_functions= 		--> disable_functions=
   ;memory_limit=540Mb 		--> memory_limit=2G
   ;max_input_vars = 900 	--> max_input_vars = 200000
   ;max_input_time=60		--> max_input_time=600
   ;max_execution_time=1200	--> max_execution_time=3600
   ;disable_functions=          --> disable_functions = exec,execl,system,passthru,shell_exec,set_time_limit,escapeshellarg,escapeshellcmd,proc_close,ini_alter,proc_open,dl,popen,show_source,posix_getpwuid,getpwuid,posix_geteuid,posix_getegid,posix_getgrgid,open_basedir,safe_mode_include_dir,pcntl_exec,pcntl_fork,putenv,proc_get_status,proc_nice,proc_terminate,pclose,virtual,openlog,popen,pclose,virtual,openlog,escapeshellcmd,escapeshellarg,dl,show_source,symlink,eval,mail
   
   
2) ALTERAR O ARQUIVO httpd.conf NO BOTÃO Config DO SERVIDOR Apache no XAMPP.
   Mudar o bloco <Directory /> para:
   <Directory />
       Options Indexes FollowSymLinks
       AllowOverride All
       Require all granted
   </Directory>
 
CRIAR UM DOMINIO LOCAL PARA USO DA LOJA NA MÁQUINA
--------------------------------------------------
 
 1) ABRIR O EXPLORADOR DE ARQUIVOS DO WINDOWS
 2) EDITAR O ARQUIVO: C:\Windows\System32\drivers\etc\hosts (A EDIÇÃO SÓ É POSSIVEL COMO ADMINISTRADOR)
 3) COLOQUE NO FIM DO ARQUIVO O SEGUINTE CAMINHO: 127.0.0.1           local.nomedaloja.com.br
    Quando for executar a loja, será usado esse dominio. Ex: http://local.nomedaloja.com.br/magento2
 4) SALVE O ARQUIVO COMO ADMINISTRADOR.

===========================================

CRIAR O VÍNCULO COM COMUNIDADE MAGENTO ATRAVÉS DO MARKETPLACE
-------------------------------------------------------------

1) CRIAR UMA CONTA NA PLATAFORMA DE MARKETPLACE DO MAGENTO NO SITE: https://marketplace.magento.com/
   OPÇÃO: Sing In no canto superior da tela da plataforma
   
   [SE NÃO POSUIR UMA CONTA CADASTRADA]
   
   New Users - Botão Register
   Digite:  First Name, Last Name, Email Address
   Escolha: My Company Primarily - (Develops Magento extensions)
            Ny Role              - (Technical/developer) 
            Country              - (Brazil)
   Digite:  Password, Confirm Password
   Clica:   Não sou um robô
   Selecione: By supplying my contact information, I authorize the Adobe family of companies to contact me via personalized communications about Adobe's products and services. See our Privacy Policy for more details or to opt-out at any time.
   Clique no botão: Submit
   Observação: Provavelmente receberá uma confirmação no e-mail digitado. Verifique na caixa de lixo do e-mail.
   
2) ENTRAR CONTA NA PLATAFORMA DE MARKETPLACE DO MAGENTO NO SITE: c
   OPÇÃO: Sing In no canto superior da tela da plataforma
 
   [SE POSUIR UMA CONTA CADASTRADA]
 
   ENTRAR NA OPÇÃO Registered Users 
   Digite: Email, Password
   Clica:  Login
   Clicar: Em cima do seu nome no canto superior direito da tela do MarketPlace e escolha My Profile
   Na aba Marketplace/Products clicar em Access Keys
   Na aba Magento 2 clique no botão Create A New Access Key
   Abrirá uma tela pop-up: Digite um nome para essa chave de acesso e em seguida clique em OK
   A chave de acesso será apresentada na tela anterior através dos campos: Name, Access Keys, Status, Actions
   No campo Access Keys você terá: Public Key e Private Key que serão usados na instalação da loja
   Note que ao lado das chaves tem um botão Copy que servirá para você copiar as chaves quando solicitadas na instalação
   por esse motivo, mantenha essa página aberta para que possa fazer a cópia depois. 
   A Public Key na instalação será o USERNAME e a Private Key será a PASSWORD no momento da instalação.
   No campo PASSWORD só aceitará se você copiar a Private Key através do botão Copy. No momento da cópia, o valor não
   aparecerá na tela da instalação.
 
 CRIAR O BANCO DE DADOS (MARIADB XAMPP) QUE SERÁ USADO NA LOJA
 -------------------------------------------------------------
  
 1) ABRA O Control Painel do XAMMP E CLIQUE NOS BOTÕES Start do Apache e MySQL (Os módulos devem estar verdes)
 2) ABRA O SEU NAVEGADOR DE INTERNET E DIGITE: localhost 
    OBSERVAÇÃO: A BASE DE EXECUÇÃO DO SERVIDOR APACHE NO XAMPP FICA NA PASTA: C:\xampp\htdocs    
 3) APARECERÁ A TELA PADRÃO DO Apache Friends. CLIQUE NO LINK phpMyAdmin NO CANTO SUPERIOR DIREITO DA TELA
 4) O PAINEL DE ADMINISTRAÇÃO DE BANCO DE DADOS SERÁ APRESENTADO. CLIQUE À DIREITA NO BOTÃO Novo PARA CRIAR UM NOVO BANCO DE DADOS
 5) À DIREITA NA ABA Base de Dados DIGITE O NOME DO NOVO BANCO DE DADOS: magento2 E À DIREITA DO NOME NO CAMPO SEGUINTE 
    TEM QUE ESTAR:utf8mb4_general_ci
 6) CLIQUE NO BOTÃO Criar
 7) NOTE QUE O BANCO DE DADOS É APRESENTADO À ESQUERDA NA LISTA DE BANCOS DISPONÍVEIS.
 8) DEIXE ESSA TELA DO BANCO DE DADOS ABERTA
 
  
 BAIXAR E INSTALAR A LOJA NO SERVIDOR LOCAL
 ==========================================
 
 BAIXANDO A LOJA NO SERVIDOR LOCAL
 ---------------------------------
 
 1) CLICAR NO BOTÃO Shell DO  Control Panel do XAMPP
 2) A TELA DO shell SE ABRIRÁ NA PASRA c:\xampp. MUDE PARA A PASTA htdocs DIGITANDO: cd htdocs
 3) JA DENTRO DA PASTA c:\xampp\htdocs, CRIE O PROJETO DA LOJA COM O SEGUINTE COMANDO:
     
     composer create-project --repository-url=https://repo.magento.com/ magento/project-community-edition <install-directory-name>
     php -d memory_limit=-1 composer create-project --repository-url=https://repo.magento.com/ magento/project-community-edition=2.3.6 loja --ignore-platform-reqs
     
     
     
     ONDE: <install-directory-name> É O NOME DA PASTA ONDE A LOJA SERÁ CRIADA. Ex:
     
     composer create-project --repository-url=https://repo.magento.com/ magento/project-community-edition magento2
 4)  AO TECLAR ENTER APÓS O COMANDO, A INSTALAÇÃO PEDIRÁ:
     UserName: - Vá até a tela do Marketplace já aberta no browser anteriormente e clique em Copy Public Key
                 Volte para a tela do shell e clique com o botão direito do mouse. A chave será copiada para o campo Username. Tecle Enter.
     Password: - Vá até a tela do Marketplace já aberta no browser anteriormente e clique em Copy Private Key
                 Volte para a tela do shell e clique com o botão direito do mouse. A chave será copiada para o campo Password mas não aparecerá.
                 Tecle Enter.
 5)  APARECERÁ A PERGUNTA: Do you want to store credentials for repo.magento.com in c:/Users/Arup Roy/AppData/Roaming/Composer/auth.json ? [Yn]
 6)  TECLE Y e TECLE Enter
 7)  COMECARÁ O DOWNLOAD DA LOJA. ESSE PROCESSO LEVARÁ ALGUM TEMPO. DEIXE FINALIZAR.
     ATENÇÃO: ESSA ETAPA SÓ DARÁ CERTO SE OS PROCEDIMENTOS ANTERIORES FOREM FEITOS CORRETAMENTE E CONCLUIDOS COM SUCESSO
     
 INSTALANDO A LOJA NO SERVIDOR LOCAL
 -----------------------------------
 
 1) APÓS O DOWNLOAD DA LOJA, ABRA UMA NOVA TELA DE NAVEGADOR E DIGITE: http://local.nomedaloja.com.br/magento2
 2) A TELA DE ENTRADA PARA INSTALAÇÃO DO MAGENTO SERÁ APRESENTADA. CLIQUE NO BOTÃO Agree and Setup Magento
 3) A TELA DE INSTALAÇÃO DO MAGENTO SERÁ APRESENTADA. NOTE QUE NO TOPO É APRESENTADA UMA RÉGUA COM 6 ETAPAS DA INSTALAÇÃO
    A PRIMEIRA ETAPA É A CHECAGEM DO AMBIENTE. CLIQUE NO BOTÃO Start Readiness Check.
    ATENÇÃO: NESSA ETAPA TODO O AMBIENTE TEM QUE ESTAR OK. DEVE APARECER TICKES VERDES DE OK EM TODA A CHECAGEM
             SE APARECER ALGUM ERRO (TICKES VERMELHOS), VERIFIQUE NA PREPARAÇÃO DO AMBIENTE NO TÓPICO: "ALTERAR OS ARQUIVOS DE CONFIGURAÇÃO DO XAMPP".
 4) ESTANDO O AMBIENTE TODO OK CLIQUE NO BOTÃO NEXT , A SEGUNDA ETAPA Add a Database é Apresentada. 
    OS CAMPOS SÃO APRESENTADOS:
       - Database Server Host:          localhost  (deixe como está)
       - Database Server Username:      root       (deixe como está)
       - Database Server Password:      vazia      (deixe como está)
       - Database Name:			magento    (digite magento2 que é o nome do banco de dados vazio criado anteriormente)
       - Table prefix:                  (optional) (deixe como está)
 5) CLIQUE NO BOTÃO NEXT E A TELA Web Configuration É APRESENTADA:
    OS CAMPOS SÃO APRESENTADOS:
       - Your Store Address:     http://local.nomedaloja.com.br/magento2 (deixe como está)
       - Magento Admin Address:	admin_xxxx - onde xxxx é uma extensão aleatória. (apague a extensão e deixe somente admin)
 6) CLIQUE NO BOTÃO NEXT E A TELA Customize Your Store É APRESENTADA:
    OS CAMPOS SÃO APRESENTADOS:
       - Store Time Zone: Coordinated Universal Time (UTC)  (Selecione Brasilia/São Paulo)
       - Store Default Currency: US Dollar (USD)            (Selecione Brazil Real (BRL)
       - Store Default Language: English (United States)    (Selecione Poruguese Brazil)
 7) CLIQUE NO BOTÃO NEXT E A TELA Create Admin Account É APRESENTADA
    OS CAMPOS SÃO APRESENTADOS:
       - New User Name			(Digite um nome para o administrador)
       - New Email			(Digite um email válido para o administrador)
       - New Password			(Digite uma senha para o administrador)
       - Confirm Password	        (Digite a confirmação da senha para o administrador)
 8) CLIQUE NO BOTÃO NEXT E A TELA Install É APRESENTADA E A INSTALAÇÃO COMEÇA.
    AGUARDE A INSTALAÇÃO TERMINAR. ESSE PROCESSO DEMORA ALGUNS MINUTOS.
 9) APÓS A INSTALAÇÃO A TELA Magento Admin Info É APRESENTADA
 10) NO CAMPO: Your Store Address ESTÁ O CAMINHO DA LOJA PADRÃO (front-end) COPIE O ENDEREÇO E COLE NO NAVEGADOR
     ESSE PROCESSO ABRIRÁ A TELA DA LOJA VAZIA. ISSO SIGNIFICA QUE A INSTALAÇÃO EXECUTOU COM SUCESSO.
 11) NO CAMPO: Magento Admin Address ESTÁ O CAMINHO DA PÁGINA DE ADMINISTRAÇÃO DA LOJA (back-end) COPIE E COLE NO NAVEGADOR
     A PÁGINA DA ADMINISTRAÇÃO TEM UM BUG A SER CORRIGIDO, POIS AO ABRIR NO NAVEGADOR ELA FICA ESCURA E NÃO APRESENTA A TELA DE LOGIN E SENHA.
     NESSE CASO, DEVEMOS CORRIGIR ESSE BUG PARA ACESSAR A TELA DA ADMINISTRAÇÃO.
     
 CORIGINDO O BUG PARA ACESSO À TELA DE ADMINISTRAÇÃO
 ---------------------------------------------------
  
 1) ATRAVÉS DO EXPLORADOR DE ARQUIVOS DO WINDOWS EDITE O SERGUINTE ARQUIVO:
    C:\xampp\htdocs\magento2\vendor\magento\framework\View\Element\Template\File\Validator.php
    
    procure a linha: $realPath = $this->fileDriver->getRealPath($path);
    
    e troque por $realPath = str_replace('\\','/',$this->fileDriver->getRealPath($path));
 
 2) APÓS A EDIÇÃO DO ARQUIVO, GRAVE O MESMO.
 
 ENTRANDO NA TELA DO ADMINISTRADOR
 ---------------------------------
 
 1) VOLTE NA TELA Magento Admin Info DA INSTALAÇÃO, COPIE O Magento Admin Address
 2) ABRA UMA PÁGINA DE NAVEGAÇÃO ANÔNOMA NO BROWSER. NO CHROME VOCE PODE ABRIR DE DUAS FORMAS:
    - TECLANDO: CTRL-SHIFT-N
    - PELO MENU NO CANTO SUPERIOR DIREITO: NOVA JANELA ANÔNIMA.
 3) COLE O ENDEREÇO DO ADMINISTRADOR NA TELA ANÔNIMA.
 4) A TELA DE ENTRADA NA ÁREA DE ADMINISTRAÇÃO É APRESENTADA.
    DIGITE OS CAMPOS: Username e Password. EM SEGUIDA CLIQUE NO BOTÃO Sign In.
 5) A TELA DE ADMINISTRAÇÃO É APRESENTADA.
 
  
 EXECUTAR PROCEDIMENTOS DE AJUSTE DE AMBIENTE NA LOJA
 ====================================================
 
 LIMPAR O CACHE
 --------------
 
 1) DENTRO DO PAINEL ADMINISTRATIVO NO MENU ACESSE AS SEGUINTE OPÇÃO:
    SYSTEM / Cache Management
 2) CLIQUE NO BOTÃO Flush Magento Cache
 3) AO FINALIZAR A LIMPEZA DO CACHE, SELECIONE TODAS AS LINHAS DA GRADE
 4) ESCOLHA A OPÇÃO Enable NO TOPO DA LISTA E NA SEQUENCIA CLIQUE EM Submit
    ATENÇÃO: NESSE MOMENTO NÃO EXECUTE AS OPÇÕES DE LIMPEZA DE CACHE ABAIXO DA GRADE.
             SE ALGUMA OPÇÃO DA GRADE O STATUS ESTIVER COMO INVALIDADE, SELECIONE A LINHA, 
             ESCOLHA Refresh NO TOPO DA GRADE E EM SEGUIDA CLIQUE EM Submit.
 
 REINDEXAR OS ÍNDICES DA LOJA
 ----------------------------
 
 1) ACESSE O Shell NO WAMPP NA PASTA: htdocs\magento2
 2) EXECUTE O SEGUINTE COMANDO: php bin/magento indexer:reindex

MANEIRA 1 (COMPOSER) DE INSTALAR (MAIS FÁCIL)
---------------------------------------------

 1) SE VOCÊ INSTALOU DA MANEIRA 1, PARA TESTAR INSTALAR PELA MANEIRA 2, APAGUE A PASTA:c:xampp/htdocs/magento2/app/i18n
    ACESSE O Shell NO WAMPP NA PASTA: htdocs\magento2
    EXECUTE O SEGUINTE COMANDO: php bin/magento cache:flush 
    TUDO VOLTARÁ PARA O INGLÊS
 2) ACESSE O Shell NO WAMPP NA PASTA: htdocs\magento2
 3) EXECUTE O SEGUINTE COMANDO: php composer.phar require magento2translations/language_pt_br:dev-master --ignore-platform-reqs
 4) AO TECLAR ENTER APÓS O COMANDO, A INSTALAÇÃO PEDIRÁ:
     UserName: - Vá até a tela do Marketplace já aberta no browser anteriormente e clique em Copy Public Key (Chave usada pela loja)
                 Volte para a tela do shell e clique com o botão direito do mouse. A chave será copiada para o campo Username. Tecle Enter.
     Password: - Vá até a tela do Marketplace já aberta no browser anteriormente e clique em Copy Private Key (Chave usada pela loja)
                 Volte para a tela do shell e clique com o botão direito do mouse. A chave será copiada para o campo Password mas não aparecerá.
                 Tecle Enter.
 5) APARECERÁ A PERGUNTA: Do you want to store credentials for repo.magento.com in c:/Users/Arup Roy/AppData/Roaming/Composer/auth.json ? [Yn]
 6) TECLE Y e TECLE Enter
 7) APÓS O INICIO DA INSTALAÇÃO O SISTEMA PEDIRÁ O CAMPO: Token (hidden): (TECLE ENTER) E AGUARDE O FIM DO PROCESSO QUE DEMORA UM POUCO.
 9) ASSIM QUE FINALIZAR, EXECUTE O SEGUINTE COMANDO: php bin/magento cache:clean
10) AO FIM DO PROCESSO, EXECUTE O SEGUINTE COMANDO: php bin/magento setup:static-content:deploy pt_BR -f
11) VERIFIQUE OS RESULTADOS ACESSANDO A LOJA: http://local.nomedaloja.com.br/magento2
12) VERIFIQUE OS RESULTADOS ACESSANDO A ADMINISTRAÇÃO: http://local.nomedaloja.com.br/magento2/admin 
    NOTE QUE DESSA MANEIRA TANTO O FRONT-END QUANTO O BACK-END ESTARÃO TRADUZIDOS.


MANEIRA 2 (MANUAL) DE INSTALAR

 1) BAIXAR O PACOTE OFICIAL DE TRADUÇÃO DO GIT HUB NO ENDEREÇO: https://github.com/Magento2Translations/language_pt_br 
 2) BAIXE O ARQUIVO (language_pt_br-master.zip) PARA UMA PASTA DO PC E DESCONPACTE ELE NESSA PASTA.

 6) EXECUTE O SEGUINTE COMANDO: php bin/magento cache:flush
 7) VERIFIQUE OS RESULTADOS ACESSANDO A LOJA: http://local.nomedaloja.com.br/magento2
 8) VERIFIQUE OS RESULTADOS ACESSANDO A ADMINISTRAÇÃO: http://local.nomedaloja.com.br/magento2/admin 
    NOTE QUE A TELA DE LOGIN JÁ ESTÁ EM PORTUGUÊS, POIS A MESMA AINDA FAZ PARTE DO FRONT-END

10) CLIQUE NA OPÇÃO Account Setting (nomedousuario)
12) NA TELA ESCOLHA NO CAMPO: Interface Locale: Português (Brasil) / Poruguese (Brazil)
13) DIGITE A SENHA DO USUÁRIO LOGADO NO CAMPO: Your Password
14) CLIQUE NO BOTÃO Save Account NO CANTO SUPERIOR DIREITO DA TELA. ASSIM QUE GRAVAR, NOTE QUE TODO O SITE JÁ FOI TRADUZIDO.

    OBSERVAÇÃO: IMAGINE QUE NO INICIO DA INSTALAÇÃO VOCÊ ESQUECEU DE COLOCAR QUE A LOJA ERA EM PORTUGUÊS.
                NESSE CASO, ACESSE NO MENU A ESQUERDA. Lojas/Configuração/Opções de Localidade
                NO CAMPO Localidade você coloca: Português (Brasil)

CRIAR UMA CONTA NO PAGSEGURO
============================

A CONTA DO PAGSEGURO É GRÁTIS, NÃO POSSUI TARIFA, É CRIADA UMA CONTA CORRENTE E AINDA PODE 
TER UM CARTÃO DE DÉBITO CRÉDITO SEM ANUIDADE PARA MOVIMENTAR O DINHEIRO DESSA CONTA.
ESSA CONTA SERÁ A QUE VAI SER USADA NOS MEIOS DE PAGAMENTO DENTRO DA LOJA SE OPTAR EM INSTALAR
O MÓDULO DE PAGSEGURO COMO MEIO DE PAGAMENTO.
NO MEU CASO, COMO JÁ TENHO A CONTA, ENTÃO CADASTREI O MÓDULO E ABAIXO ESTÃO AS INSTRUÇÕES 
PARA INSTALAR O MÓDULO.
A CONTA DO PAGSEGURO, EXIGIRÁ O ENVIO DE FOTOS DE ALGUNS DOCUMENTOS PARA LIBERAÇÃO DA CONTA
QUE DEVE OCORRER UM OU DOIS DIAS APÓS A SOLICITAÇÃO.

EM PRIMEIRO MOMENTO, A INSTALAÇÃO DO MÓDULO, NÃO ESTÁ ATRELADO A ABERTURA DA CONTA OU SEJA,
NÃO É NECESSÁRIO CRIAR A CONTA, A ÚNICA COISA QUE VAI ACONTECER É QUE 
NÃO DARÁ PARA TESTAR OS PAGAMENTO DO PAGSEGURO ATRAVÉS DA LOJA.

SE QUISER CRIAR A CONTA, SIGA AS SEGUINTES INSTRUÇÕES:

1) ENTRAR NO SITE DO PAGSEGURO: https://pagseguro.uol.com.br/

     NESSE CASO O IDEAL É O PRÓPRIO CLIENTE ABRIR A CONTA PARA ELE.
4) CLIQUE EM Continuar E SIGA AS INSTRUÇÕES PARA A CRIAÇÃO DA CONTA. (NÃO VOU CRIAR PORQUE JÁ TENHO UMA)

Configurações Gerais
====================

ambiente: especifica em que ambiente as transações serão feitas (produção/sandbox).
e-mail: e-mail cadastrado no PagSeguro.

token: token cadastrado no PagSeguro. (DENTRO DO´SITE DO PAGSEGURO, GERE O TOKEN EM: VENDAS ON-LINE / INTEGRAÇÕES / UTILIZAÇÕES DE API / BOTÃO GERAR TOKEN) GERE O TOKEN, COPIE E COLE NESSE CAMPO.

url de redirecionamento: ao final do fluxo de pagamento no PagSeguro, seu cliente será redirecionado automaticamente para a página de confirmação em sua loja ou então para a URL que você informar neste campo. Para ativar o redirecionamento ao final do pagamento é preciso ativar o serviço de Pagamentos via API. Obs.: Esta URL é informada automaticamente e você só deve alterá-la caso deseje que seus clientes sejam redirecionados para outro local.
url de notificação: sempre que uma transação mudar de status, o PagSeguro envia uma notificação para sua loja. O valor padrão que deve ser utilizado pelo módulo é: http://www.minhaloja.com.br/index.php/pagseguro/notification/response
Observação: Esta URL só deve ser alterada caso você deseje receber as notificações em outro local.
charset: codificação do seu sistema (ISO-8859-1 ou UTF-8).
ativar log: ativa/desativa a geração de logs.
diretório: informe o local e nome do arquivo a partir da raíz de instalação do Magento onde se deseja criar o arquivo de log. Ex.: var/log/pagseguro.log.
Por padrão o módulo virá configurado para salvar o arquivo de log em var/log/pagseguro.log.
listar transações abandonadas?: ativa/desativa a pesquisa de transações que foram abandonadas no checkout do PagSeguro.
transações -> abandonadas: permite consultar as transações que foram abandonadas nos últimos 10 dias, desta forma você pode enviar emails de recuperação de venda. O e-mail conterá um link que redirecionará o comprador para o fluxo de pagamento, exatamente no ponto onde ele parou.
habilitar recuperação de carrinho: Habilita a recuperação de carrinho do PagSeguro. (por padrão está desabilitada)
listar parcelamento: Habilita a exibição de uma listagem de parcelas na tela de visualização do produto. (Irá exibir o maior parcelamento disponível para o produto na tela de exibição do mesmo)

####################################
## COMO DEIXAR A LOJA MAIS RÁPIDA ##
####################################

1) OTIMIZAR OS REQUISITOS DO SERVIDOR E DO SISTEMA (NÃO FIZ)
   ---------------------------------------------------------
   Os requisitos de servidor e sistema podem ser considerados os dois fatores mais importantes que podem ajudá-lo 
   a aumentar a velocidade do seu site Magento 2.
   Requisitos do servidor: Como o Magento é uma loja bastante poderosa, em vez de hospedagem compartilhada para o 
   Magento , você deve hospedar sua loja Magento em um servidor dedicado ou hospedagem VPS .
2) ATUALIZE A VERSÃO MAIS RECENTE DO MAGENTO 2 (ESTOU NA VERSÃO 2.3.4)
   -------------------------------------------------------------------
   Quando se trata de otimizar a velocidade do Magento, uma das maneiras mais simples que você não pode esquecer de 
   usar é atualizar a versão mais recente. Sempre que o Magento lançar uma nova versão, o desempenho, assim como 
   outros ajustes, serão melhorados significativamente. Conseqüentemente, é altamente recomendável que você atualize 
   a versão da sua loja para a mais recente, a fim de melhorar a velocidade do site da sua loja.
3) ATIVAR O CACHE DE VERNIZ
   ------------------------
   Existem várias práticas que precisam ser implementadas para acelerar o site Magento, incluindo o cache.
   Altere em: Lojas > Configuração > Avançado > Sistema > Aba: Cache de Página Inteira 
              Campo: Cache Aplicação - Escolha: Cache Varnish (Recomendada)
4) CONFIGURAR O MEMCACHED (NÃO FIZ)
   --------------------------------
   Outro método para acelerar as lojas Magento 2 é diminuir a carga do banco de dados. E a maneira de fazer isso é 
   configurar o Memcached, que é um sistema de armazenamento em cache de objetos de memória distribuída de código 
   
   - instalar o Memcached - veja em: https://gist.github.com/tomysmile/339a6f04b1570a0803d8facdcf2a27cd
   
   aberto e gratuito. Como fazer:
    - Abra o arquivo: C:\xampp\htdocs\magento2\app\etc\env.php
    - Localize o trecho abaixo:
      'session' => [
              'save' => 'files'
      ],
    - troque por:
      'session' =>
          array [
            'save' => 'memcached',
            'save_path' => '<memcache ip or host>:<memcache port>'
      ],
    - apague o conteúdo das pastas: var/cache, var/page_cache e var/session
    - Navegue entre as páginas da loja e adminitrativo. Se não der problema a configuração deu tudo certo.
5) ATIVAR CATEGORIAS E PRODUTOS SIMPLES
   ------------------------------------
   Um dos motivos mais comuns que tornam o site Magento 2 mais lento é a velocidade de leitura do banco de dados. 
   Para corrigir esse problema, você pode habilitar o Flat Catalog em:
    - Lojas > Configuração > Catálogo - Aba Frente de Loja - Habilitar os seguintes campos:
      - Usar Categoria do Catálogo Flat : Sim
      - Usar Produto do Catálogo Flat: Sim
6) OTIMIZAR JAVASCRIPT E CSS
   -------------------------
   Outro motivo que leva à baixa velocidade do Magento 2 é o Javascript e CSS.    
   A seguir, está o guia para minimizar o Javascript e CSS , a fim de aumentar a velocidade do Magento 2:
   - Lojas > Configuração > Avançado > Desenvolvedor 
        - Aba Definições de JavaScript - Habilitar os campos:
          - Combinar Ficheiros JavaScript: Sim
          - Ativar Combinação de Ficheiros: Sim
          - Reduzir Dimensão de Ficheiros JavaScript
        - Aba Configurações CSS - Habilitar os seguintes campos:
          - Mesclar arquivos CSS: Sim
          - Reduzir dimensão dos Ficheiros CSS: Sim
7) CONFIGURAR A REDE DE ENTREGA DE CONTEÚDO
   ----------------------------------------
   Você também pode configurar o Content Delivery Network para melhorar a velocidade da sua loja Magento em:
   - Lojas > Configuração > Geral > Web - 
       - Aba URLs Base - Preencha o endereço da loja no campo:
        - URL Base: http://sualoja.com.br
       - Aba URLs Base (Seguro) - Preencha o endereço da loja no campo:
        - URL Base: https://sualoja.com.br
8) USE UM BOM TEMA
   ---------------
   É altamente recomendável usar um tema Magento mais rápido e leve, pois pode ajudar a melhorar 
   significativamente a velocidade de carregamento do Magento 2.
   Melhores temas: https://www.mageplaza.com/magento-2-themes/
9) ESCOLHA EXTENSÕES
   Vários usuários tendem a usar extensões sem verificar ou confirmar sua segurança. Se contiver erros, 
   sua velocidade do Magento definitivamente será afetada. É por isso que você sempre deve usar uma 
   extensão de terceiros sem bugs para sua loja Magento 2.   
   Melhores extensões: https://www.mageplaza.com/magento-2-extensions/
10) OTIMIZAR IMAGENS
    ----------------
    No site da sua loja, existem inúmeras imagens de produtos. Isso pode diminuir a velocidade do 
    Magento se essas imagens não forem otimizadas corretamente. Aqui estão alguns pontos-chave na otimização 
    de imagem:
    - As imagens precisam ser compactadas.
    - O logotipo da sua loja deve estar no formato PNG ou SVG.
    - A imagem do produto deve estar no formato JPEG.

#######################
## PASSOS NO LOCAWEB ##
#######################

1) CONTRATAR UM SERVIÇO DE HOSPEDAGEM VPS.
2) ADICIONAR A LINHA DE CÓDIGO ABAIXO NO FINAL DO ARQUIVO php.ini
   suhosin.executor.include.whitelist = phar
3) COLOCAR O ARQUIVO php.ini NA PASTA public_html
4) ACCESSAR O FTP ATRAVÉS DO PUTTY
5) TROCAR A VERSÃO SO PHP COM O SEGUITE COMANDO:
  locaweb-change-php 7.2
6) EXECUTAR A INSTALAÇÃO DO COMPOSER COM O SEGUINTE COMANDO:
   curl -sS https://getcomposer.org/installer | php7.3 -c /home/tedolls1/.user.ini
7) VERIFICAR OS COMANDOS DO COMPOSER APÓS A INSTALAÇÃO
   php7.2 -c /home/tedolls1/.user.ini composer.phar
8) PROCESSO DE INSTALAÇÃO DA LOJA
9) EXECUTAR OS COMANDOS DE PERMISSÃO DE PASTAS E ARQUIVOS:
   find * -type d -print0 | xargs -0 chmod 0755
   find . -type f -print0 | xargs -0 chmod 0644
